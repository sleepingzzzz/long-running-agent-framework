# 任务管理指南

## 概述

`tasks.json` 中的 31 个任务只是**示例**，展示视频中的"AI漫剧生成工具"项目。

**你应该根据自己的实际需求，创建自己的任务列表！**

## 快速开始

### 方法 1: 使用任务管理工具（推荐）

```bash
cd E:\code\qclaw-agent-framework

# 查看当前任务
node scripts/task-manager.js list

# 添加新任务
node scripts/task-manager.js add "实现登录功能" "创建用户登录页面和API" --priority high --hours 2

# 删除任务
node scripts/task-manager.js remove 5

# 更新任务
node scripts/task-manager.js update 3 priority=high

# 清空所有任务（从零开始）
node scripts/task-manager.js clear

# 重置所有任务状态
node scripts/task-manager.js reset
```

### 方法 2: 使用模板

```bash
# 使用空白模板（从零开始）
copy templates\tasks-empty.json tasks.json

# 或使用示例模板
copy templates\tasks-example-blog.json tasks.json
```

### 方法 3: 直接编辑

直接用文本编辑器打开 `tasks.json` 修改。

## 任务字段说明

```json
{
  "id": 1,                      // 任务ID（唯一）
  "title": "任务标题",           // 简短标题
  "description": "详细描述",     // 具体要做什么
  "status": "pending",           // 状态: pending/in_progress/completed/failed
  "priority": "high",            // 优先级: high/medium/low
  "estimated_hours": 2,          // 预计耗时（小时）
  "created_at": "2026-03-20",    // 创建日期
  "completed_at": null,          // 完成日期
  "notes": ""                    // 备注
}
```

## 任务设计建议

### ✅ 好的任务

- **具体明确**: "实现用户登录API" ✓
- **可独立完成**: 一个任务2小时内能完成
- **可测试验证**: 完成后能明确判断是否成功
- **原子性**: 不依赖其他未完成任务

### ❌ 避免的任务

- **太模糊**: "做后端" ✗
- **太大**: "完成整个项目" ✗
- **无法验证**: "优化代码"（没有标准）✗

## 任务拆分技巧

### 从需求到任务

**原始需求**: "做一个电商网站"

**拆分为任务**:
1. 项目初始化（目录结构、Git、基础配置）
2. 数据库设计（用户、商品、订单表）
3. 用户系统（注册/登录/个人中心）
4. 商品模块（列表/详情/搜索）
5. 购物车功能
6. 订单系统（创建/支付/查询）
7. 管理后台
8. 前端页面（首页/商品页/购物车/订单）
9. 支付集成
10. 测试部署

### 任务粒度

- **太小**: 每个任务 < 15分钟 → 任务太多，管理成本高
- **合适**: 每个任务 30分钟-2小时 → 平衡
- **太大**: 每个任务 > 4小时 → 难以估算和验证

## 动态调整

项目进行中可以随时调整：

- **添加新任务**: 发现遗漏的功能
- **删除任务**: 需求变更
- **修改任务**: 调整范围或优先级
- **重新排序**: 调整执行顺序

## 最佳实践

1. **先规划再执行**: 花10分钟想清楚任务列表，省1小时返工
2. **从简单开始**: 先做能快速完成的任务建立信心
3. **保持灵活**: 任务列表是活的，随时调整
4. **记录经验**: 在 notes 中记录遇到的问题

---

**记住**: 任务数量不重要，重要的是清晰、可执行、可验证！
