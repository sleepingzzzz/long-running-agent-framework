# 架构说明

长效运行智能体框架的系统架构和设计。

## 概述

框架由三个主要组件组成：

1. **任务存储** (`tasks.json`): 持久化任务定义和状态
2. **执行引擎** (`scripts/`): 任务选择和执行逻辑
3. **智能体接口** (`WORK_SPEC.md`): AI 智能体工作规范

## 数据流

### 1. 任务选择

```javascript
// 获取可用任务（待处理 + 依赖满足）
const available = tasks.filter(t => 
  t.status === 'pending' &&
  t.depends_on.every(depId => 
    tasks.find(d => d.id === depId).status === 'completed'
  )
);

// 按优先级排序
const sorted = available.sort((a, b) => 
  priorityWeight[b.priority] - priorityWeight[a.priority]
);

// 选择最高优先级
const nextTask = sorted[0];
```

### 2. 任务执行

```javascript
// 更新状态为进行中
updateTaskStatus(task.id, 'in_progress');

// 使用任务规范生成子代理
spawnAgent({
  task: buildAgentPrompt(task),
  timeout: 1800 // 30 分钟
});

// 等待完成
// 更新状态为完成/失败
updateTaskStatus(task.id, result.status);
```

### 3. 状态持久化

所有状态更改立即持久化到 `tasks.json`：

```json
{
  "project": "名称",
  "last_updated": "2024-01-01T00:00:00Z",
  "total_tasks": 10,
  "completed": 3,
  "in_progress": 1,
  "pending": 6,
  "tasks": [...]
}
```

## 依赖解析

框架使用简单的依赖图：

```
任务 A (depends_on: [])
  └── 任务 B (depends_on: [A])
        └── 任务 C (depends_on: [B])
              └── 任务 D (depends_on: [B, C])
```

执行顺序: A → B → C → D

## 扩展点

### 自定义任务类型

添加自定义字段到任务：

```json
{
  "id": 1,
  "title": "自定义任务",
  "custom_field": "值"
}
```

### 自定义脚本

在 `scripts/` 目录创建新脚本：

```javascript
const { loadTasks, saveTasks } = require('./task-manager');

// 你的自定义逻辑
```

### 钩子（计划）

未来版本计划：
- `pre-execute`: 任务执行前运行
- `post-execute`: 任务完成后运行
- `on-fail`: 处理任务失败

## 性能考虑

- **文件 I/O**: 所有状态更改立即写入磁盘
- **内存**: 每次操作加载一次任务列表
- **并发**: 每个框架实例单线程执行

## 安全

- 文件操作限制在框架目录
- 核心脚本无网络访问
- Git 提交需要正确配置
