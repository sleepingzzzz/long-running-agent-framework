# 自动化任务生成指南

## 🎯 目标

你只需要**描述需求**，AI 自动分析并生成详细的任务列表。

## 🚀 快速使用

### 方式 1: 一句话生成（推荐）

在 QClaw 中直接说：

```
请为我的项目生成开发任务列表。

项目名称: 个人博客系统
项目描述: 支持Markdown的写作和发布的博客平台
需求: 
- 用户可以注册登录
- 支持Markdown编辑和预览
- 文章可以分类和标签
- 支持评论功能
- 响应式设计

请分析需求，生成详细的开发任务列表，保存到 E:\code\qclaw-agent-framework\tasks.json
```

### 方式 2: 使用预设模板

```
基于以下模板生成任务：

项目类型: [电商网站 / 社交应用 / 管理后台 / 工具类应用 / 自定义]
核心功能: [描述主要功能]
技术栈: [React/Node.js/PostgreSQL 等]

请生成对应的开发任务列表。
```

### 方式 3: 从需求文档生成

```
请分析以下需求文档，生成开发任务：

[粘贴你的需求文档]

要求：
1. 拆分为可执行的任务（每个2小时内）
2. 包含前端、后端、数据库、测试
3. 标注优先级和预估时间
4. 保存到 E:\code\qclaw-agent-framework\tasks.json
```

## 📝 AI 生成任务的标准

当你让 AI 生成任务时，它会按照以下标准：

### 任务粒度
- ✅ 每个任务 0.5-2 小时
- ✅ 可独立完成
- ✅ 有明确的完成标准

### 任务分类
1. **项目初始化** (1-2个任务)
   - 环境配置
   - 项目结构

2. **数据库设计** (1-3个任务)
   - Schema设计
   - 迁移脚本
   - 种子数据

3. **后端开发** (根据复杂度)
   - API设计
   - 业务逻辑
   - 认证授权

4. **前端开发** (根据复杂度)
   - 页面开发
   - 组件开发
   - 状态管理

5. **测试部署** (2-3个任务)
   - 单元测试
   - 集成测试
   - 部署配置

## 🎨 示例对话

### 示例 1: 简单工具

**你:**
```
我想做一个URL短链服务，输入长链接生成短链接，点击短链接跳转。
技术栈用 Node.js + Express + MongoDB，前端用简单HTML。
请生成任务列表。
```

**AI 会生成类似:**
1. 项目初始化和依赖安装
2. 设计URL映射数据库Schema
3. 实现短链接生成API
4. 实现短链接跳转API
5. 创建前端输入页面
6. 部署和测试

**你:**
```
做一个在线协作白板，类似Miro。
功能：
- 多用户实时协作
- 支持便签、画笔、图形
- 可以保存和分享白板
- 支持摄像头和语音

技术栈：React + Node.js + Socket.io + PostgreSQL
```

**AI 会生成:**
- 15-25个详细任务

## ✏️ 生成后修改

AI 生成任务后，你可以：

### 在 QClaw 中直接修改
```
请修改任务3的描述，添加：需要支持手机号登录

删除任务7和8，合并到任务6

在任务5之后添加新任务：实现短信验证码功能
```

### 使用命令行工具
```bash
cd E:\code\qclaw-agent-framework

# 修改任务
node scripts/task-manager.js update 3 description="新的描述"

# 删除任务
node scripts/task-manager.js remove 7

# 添加任务
node scripts/task-manager.js add "新任务" "描述" --priority high
```

### 直接编辑文件
打开 `tasks.json` 直接修改。

## 🔄 迭代优化

如果第一次生成的任务不满意：

```
请重新生成任务列表，要求：
1. 任务粒度更细一些
2. 添加更多测试相关的任务
3. 把前端任务拆分为页面和组件两部分
```

## 📊 任务数量建议

| 项目规模 | 建议任务数 | 预计开发时间 |
|---------|-----------|-------------|
| 简单工具 | 5-10个 | 1-2天 |
| 小型应用 | 10-20个 | 1-2周 |
| 中型系统 | 20-40个 | 1-2月 |
| 大型平台 | 40+个 | 2月+ |

## 💡 最佳实践

1. **先粗后细**: 先生成大致任务，再细化关键部分
2. **预留缓冲**: 总预估时间 × 1.5 = 实际时间
3. **优先级排序**: 先做核心功能，再做边缘功能
4. **持续调整**: 开发中发现遗漏随时添加
